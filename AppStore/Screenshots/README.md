# Automated App Store Screenshots

Blip's App Store screenshots are generated by Fastlane from the app's real SwiftUI views. A debug-only router selects the existing Scan, History, Settings, Company Profiles, or Privacy view without recreating those screens. The router is enabled only when the app is launched with `--app-store-screenshot <scene>` and is removed from Release builds by `#if DEBUG`.

The scanner scene is also the real `ScannerPresentation`. Because iOS Simulator has no camera feed, only its `AVCaptureSession` preview is replaced by a non-identifying demo warehouse image with a generated barcode. The real scanner navigation, flashlight control, overlay, and zoom indicator remain visible. Production camera code and the keyboard-to-scanner-to-return workflow are unchanged.

## Generate all screenshots

```sh
fastlane ios screenshots
```

The lane:

1. Builds and locally signs Blip for iOS Simulator so the real app-group state shared with the keyboard extension remains available.
2. Uses the newest available `iPhone 17 Pro Max` and `iPad Pro 13-inch (M5)` simulators.
3. Launches each screenshot scene in English and German.
4. Writes App Store-ready PNGs to `AppStore/Screenshots/final/<locale>/<family>`.
5. Builds four raw review contact sheets in `AppStore/Screenshots/previews`.
6. Composes localized App Store marketing images around the real captures in `AppStore/Screenshots/composed` and builds contact sheets in `AppStore/Screenshots/composed-previews`.

The lane shuts down all other simulators before booting each target and shuts the target down after its family is captured, so iPhone and iPad Simulator never run together.

Override a simulator when needed:

```sh
BLIP_SCREENSHOT_IPHONE_UDID=<udid> BLIP_SCREENSHOT_IPAD_UDID=<udid> fastlane ios screenshots
```

The lane keeps completed PNGs, so an interrupted run resumes automatically. Use `BLIP_SCREENSHOT_CLEAN=1` to regenerate the entire set, or `BLIP_SCREENSHOT_FORCE=1` to overwrite individual existing outputs during a full run.

The final folders use App Store Connect locale identifiers:

- `en-US/iphone`
- `en-US/ipad`
- `de-DE/iphone`
- `de-DE/ipad`
